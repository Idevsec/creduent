# Creduent Foundation

**Type:** Open Protocol Stewardship Organization  
**Mission:** To maintain the Creduent Protocol as a neutral, open, and enduring standard for AI agent identity and trust.

---

## 1. About the Creduent Foundation

The Creduent Foundation is the independent governance body responsible for stewarding the Creduent Protocol. It was established to ensure that Creduent remains an open standard, not a proprietary product, as it grows from a reference implementation to universal infrastructure.

The Foundation does not develop commercial products. It:
- Maintains the protocol specification and standards track.
- Operates the governance process described in [GOVERNANCE.md](GOVERNANCE.md).
- Holds the Creduent trademark and ensures fair licensing.
- Represents the protocol in external standardization bodies (IETF, IANA, W3C).
- Coordinates security disclosure and response.

## 5. Working Groups

### 5.1 Protocol Working Group (Active)

Responsible for all standards-track documents (CREDUENT-001 through CREDUENT-005 and future CIPs).

**Scope:** agent.json schema, attestation format, registry API, URI resolution, federation model.  
**Meeting cadence:** Monthly, async via GitHub Discussions.  
**Current Chair:** *(Working Group Lead)*

### 5.2 Security Working Group (Forming)

Responsible for security policy, vulnerability disclosure, and security reviews of CIPs.

**Scope:** Threat modeling, cryptographic algorithm selection, SSRF/injection protections.  
**Status:** Accepting members. Open an issue titled "Security WG Application" to join.

## 7. Trademark Policy

"Creduent" and the Creduent logotype are trademarks of the Creduent Foundation (pending registration).

**You MAY:**
- State that your software implements the Creduent Protocol.
- Use the name "Creduent" to refer to the open protocol.
- Build and distribute SDKs, tools, and services that implement the protocol.

**You MAY NOT:**
- Use "Creduent" as the primary name of a competing registry or attestation service.
- Imply Foundation endorsement of your implementation without written permission.
- Register Creduent-prefixed domain names that could cause confusion with official Foundation resources.

For trademark questions, contact `legal@creduent.org`.
