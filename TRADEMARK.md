# Creduent™ Trademark Policy

This document outlines the policy of **IDevSec** regarding the use of the **Creduent** name, wordmark, logo, and brand assets (collectively, the "Trademarks").

---

## 1. Core Philosophy
Creduent is an open-source identity and trust protocol. While the underlying source code of our SDKs is licensed under the permissive Apache License 2.0, the **Creduent name and brand** are proprietary assets of IDevSec. 

To ensure the security of the network and protect developers and users from phishing or brand confusion, we strictly control the use of our Trademarks.

---

## 2. Permitted Uses (No Permission Required)
You may use the Creduent name without explicit written permission under the following circumstances:
- **Nominal Reference**: Referencing the Creduent Protocol or project in blog posts, technical discussions, research papers, and documentation (e.g., *"Our agent supports the Creduent protocol"*).
- **Dependency/Integration**: Describing software integrations (e.g., *"Plugin for Creduent"* or *"Creduent driver for AutoGen"*).
- **Personal and Community Activities**: Describing non-commercial meetups, community forums, or interest groups (e.g., *"Creduent Developer Meetup Paris"*).

---

## 3. Restricted Uses (Written Permission Required)
You **must** obtain written permission from IDevSec for any of the following uses:
- **Commercial Hosted Services**: Offering a public hosted, managed, or SaaS identity registry, attestation service, or gateway under the "Creduent" name (e.g., you may not launch *"AWS Creduent Directory Service"* or *"Creduent Hosting"*).
- **Forks and Redistribution**: If you fork the Creduent code and redistribute it with modifications, you **must rename the product** to something distinct. You may not call your fork "Creduent" or "Creduent Registry".
- **Product Names / Domain Names**: Using "Creduent" in the name of your commercial product, company, or domain name (e.g., `creduent-cloud.com` is prohibited).
- **Certification/Endorsement**: Using the Trademarks to imply certification, endorsement, or affiliation with IDevSec.

---

## 4. How to Ask for Permission
If you wish to use the Trademarks in a restricted scenario, please contact:
**brand-licensing@idevsec.com**

Please include:
1. Your organization name and contact information.
2. The specific Trademarks you wish to use.
3. A detailed description of the proposed use (with mocks or URLs if applicable).
